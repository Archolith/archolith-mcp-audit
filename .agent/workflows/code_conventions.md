# Code Conventions — archolith-mcp-audit

## Style

- **Indent**: 4 spaces, max 120 chars line length
- **Python version**: 3.11+
- Use builtin generics (`list`, `dict`) and `X | Y` unions, not `typing.List`/`Optional`
- Use `%s`-style lazy formatting for loggers, not f-strings
- Use `from __future__ import annotations` in modules with forward references

## Naming

| Element | Convention | Example |
|---------|------------|---------|
| Modules | snake_case | `waste_detector.py`, `schema_estimator.py` |
| Classes | PascalCase | `SessionData`, `WasteFinding`, `ServerReport` |
| Functions | snake_case | `attribute_tool()`, `count_tokens()` |
| Constants | UPPER_SNAKE_CASE | `MIN_BASE64_LENGTH`, `NUL_BYTE_THRESHOLD` |
| Config files | snake_case.json | `server_mapping.json`, `schema_catalog.json` |

## File Organization

```
archolith_mcp_audit/
├── __init__.py              # Package root, version
├── __main__.py              # python -m entry point
├── cli.py                   # Argparse CLI
├── attributor.py            # Tool → server mapping
├── tokenizer.py             # tiktoken integration
├── waste_detector.py        # 6 waste pattern detectors
├── report.py                # Report generation (text/JSON/markdown)
├── schema_estimator.py      # MCP schema token cost
├── comparator.py            # Before/after comparison
├── mcp_server.py            # In-session MCP audit tool
├── accumulator.py           # Live session accumulator
├── data/
│   ├── server_mapping.json  # Canonical server name mapping
│   └── schema_catalog.json  # Cached tool schema sizes
└── extractors/
    ├── __init__.py
    ├── base.py              # SessionData dataclass + shared interface
    ├── claude.py            # Claude JSONL extraction
    ├── codex.py             # Codex JSONL extraction
    └── opencode.py          # OpenCode SQLite extraction
```

## Testing

- **Framework**: pytest 8+
- **Location**: `tests/` at project root
- **Naming**: `test_<module>.py` — one test file per source module
- **Run**: `pytest` from project root
- **Lint**: `ruff check .`

### Test categories

| Category | Naming | What it covers |
|----------|--------|----------------|
| Unit | `test_<module>.py` | Single module logic in isolation |
| Integration | `test_extractors.py` | Full extraction from real-ish session data |
| Report | `test_report.py` | Output formatting and content |

### Fixtures

- Use `tests/fixtures/` for sample session data (small JSONL/SQLite files)
- Keep fixtures under 100KB — extract real patterns, not real data
- Never include real secrets or tokens in fixtures

## Data Files

- `data/server_mapping.json` — seeded with known workspace servers, extensible
- `data/schema_catalog.json` — generated by `--refresh-schemas`, committed
- Both loaded at startup, cached for session lifetime
