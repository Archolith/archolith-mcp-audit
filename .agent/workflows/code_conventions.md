# Code Conventions вЂ” archolith-mcp-audit

## Style

- **Indent**: 4 spaces, max 120 chars line length
- **Python version**: 3.11+
- Use builtin generics (`list`, `dict`) and `X | Y` unions, not `typing.List`/`Optional`
- Use `%s`-style lazy formatting for loggers, not f-strings
- Use `from __future__ import annotations` in modules with forward references

## Naming

| Element | Convention | Example |
|---------|------------|---------|
| Modules | snake_case | `waste_detector.py`, `schema_estimator.py` |
| Classes | PascalCase | `SessionData`, `WasteFinding`, `ServerReport` |
| Functions | snake_case | `attribute_tool()`, `count_tokens()` |
| Constants | UPPER_SNAKE_CASE | `MIN_BASE64_LENGTH`, `NUL_BYTE_THRESHOLD` |
| Config files | snake_case.json | `server_mapping.json`, `schema_catalog.json` |

## Testing

- **Framework**: pytest 8+
- **Location**: `tests/` at project root
- **Naming**: `test_<module>.py` вЂ” one test file per source module
- **Run**: `pytest` from project root
- **Lint**: `ruff check .`

### Test categories

| Category | Naming | What it covers |
|----------|--------|----------------|
| Unit | `test_<module>.py` | Single module logic in isolation |
| Integration | `test_extractors.py` | Full extraction from real-ish session data |
| Report | `test_report.py` | Output formatting and content |

### Fixtures

- Use `tests/fixtures/` for sample session data (small JSONL/SQLite files)
- Keep fixtures under 100KB вЂ” extract real patterns, not real data
- Never include real secrets or tokens in fixtures

## Data Files

- `data/server_mapping.json` вЂ” seeded with known workspace servers, extensible
- `data/schema_catalog.json` вЂ” generated by `--refresh-schemas`, committed
- Both loaded at startup, cached for session lifetime
